Project information

  • Client: The Indian Navy
  • Project date: 06/08/2021

Overview

The project intends to digitize the entire process in form of a web application that all stakeholders can use to view and add updates in real time as they participate in the procurement process.

Problem/Challenge

The current procurement process is long drawn one, with no timely real time updates available to stakeholders and users about the availability of spares or even the process of procurement. This caused following challenges :

  • No real time availability of info on the equipment and its spares in the warehouse.
  • Lack of ready information on the stage of the procurement.
  • No consolidated place to store all procurement related docs.
  • Non availability of info on person/process causing delay in procurement.
  • No ready availability of bird eyeâ€™s view of the budget and the consumption by the spares.
  • Planning for spare procurement is difficult.

Solution

To acknowledge the problems faced by Navy System, the solution of web application was introduced :

  • Develop a web application that is deployed onto the secure NUD.
  • This web application would allow all users to view the spare position and the authorized ones to make additions/edits etc.
  • Will let all users from different directorates on different projects simultaneously use the system for real time updates.
  • Will allow users to add new projects, add/edit budget, and equipment associated with the project.
  • Will allow users to add/edit relevant documents related to the procurement process.
  • Application should send appropriate notifications automatically to the concerned directorates in the procurement process.
  • The upload of documents should be verified by the concerned directorate automatically with option to approve/return with comments.
  • View the comments while approving/rejecting the documents uploaded.
  • Search for projects, equipment and the associated documents.
